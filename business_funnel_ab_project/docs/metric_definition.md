# 指标口径说明

## 1. 漏斗步骤

| 步骤 | event_name | 业务含义 |
|---|---|---|
| 访问首页 | visit_home | 用户进入业务页面 |
| 浏览商品 | view_product | 用户查看商品/服务详情 |
| 加入购物车 | add_cart | 用户产生购买意向 |
| 提交订单 | submit_order | 用户进入支付前一步 |
| 支付成功 | pay_success | 用户完成转化 |

## 2. 转化率

### 分步转化率

当前步骤去重用户数 / 上一步去重用户数

例：

加购转化率 = 加购人数 / 商品浏览人数

### 总转化率

当前步骤去重用户数 / 首页访问去重用户数

例：

支付总转化率 = 支付成功人数 / 首页访问人数

## 3. A/B 测试

### 实验设计

- control：原始结算流程
- treatment：领券提醒 + 结算页简化

### 核心假设

- H0：treatment 组支付转化率 = control 组支付转化率
- H1：treatment 组支付转化率 ≠ control 组支付转化率

### 判断规则

- p-value < 0.05：认为两组差异有统计显著性
- p-value >= 0.05：不能证明策略带来显著提升

## 4. 注意

面试中不要只说“做了图表”。这个项目的重点是：

1. 先定义业务指标；
2. 再用 SQL 稳定取数；
3. 然后用 Python 做分析和实验检验；
4. 最后给出能落地的策略建议。
