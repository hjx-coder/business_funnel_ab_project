# 简历项目描述

## 项目名称

基于 SQL + Python 的业务转化漏斗分析与策略实验评估项目

## 技术栈

SQL、Python、Pandas、SQLite、A/B 测试、Streamlit、Plotly

## 简历写法：应届生版本

- 基于用户行为日志构建转化漏斗分析项目，设计用户维表、行为事件事实表、订单事实表和实验分组表，覆盖访问、浏览、加购、提交订单、支付成功等核心链路。
- 使用 SQL 编写漏斗转化、渠道拆分、日趋势和实验指标查询脚本，计算分步转化率、总转化率、ARPU、GMV 等指标，定位不同渠道的转化差异。
- 使用 Python 对 A/B 实验结果进行双样本比例 Z 检验，比较原始结算流程与“领券提醒 + 结算页简化”策略下的支付转化率差异，输出 lift、p-value 和置信区间。
- 使用 Streamlit 搭建分析看板，展示整体漏斗、渠道转化、每日趋势和实验评估结果，并基于数据结论提出灰度放量与低转化渠道优化建议。

## 面试口述版本

这个项目模拟的是一个线上业务从访问到支付的完整转化链路。我先把数据设计成四张表：用户维表、行为事件表、订单表和实验分组表。然后用 SQL 计算访问、浏览、加购、提交订单、支付成功各步骤的人数和转化率，再按渠道拆分，判断哪些渠道转化更弱。最后我用 Python 做 A/B 测试，比较原流程和新策略的支付转化率差异，通过 p-value 判断提升是否具有统计显著性，而不是只看表面转化率变化。

## 不建议写法

不要写：

- 精通 A/B 测试；
- 负责公司核心增长策略；
- 通过项目提升公司 GMV xx%；
- 搭建企业级实时数仓。

原因：这些说法太大，容易被面试官追问穿。
